# Modbus глазами чайника

## Описание
Случайно узнал, что существуют протокол modbus и язык C#.
Специально решил на всем этом поучиться.
А чайник был выбран как наглядный прибор.

## Дальнейший план
1. ~~Сделать интерфейсы для модбас-клиентов и модбас-протоколов.~~
2. ~~Расширить функционал чайника(установка максимальной температуры).~~
3. Подключить еще одно устройство(кондиционер? увлажнитель воздуха?).
4. Вынести логику "инициализации" устройств в отдельный класс.
5. Добавить другие модбасы: аски, рту.

## Для размышлений
1. Увлажнитель воздуха можно раздробить по квартире(кухня, гостиная, спальня) и также расширить управление. И режимы придумать. А лучше еще и устройство подключить
2. Испытать вариант паттерна Команда, тогда видимо все нужно будет переписывать, а точнее из модбас-устройств доставать логику и раскидывать по командам. Тогда не нужно выбирать устройство, а потом команды, а просто сделать выбор из доступных команд.
3. Конечно же мок и модбас-устройство должны обладать схожими свойствами. Поэтому нужно для "устройств" сделать какую-то отдельную статическую структуру с параметрами типа (ячейки памяти, мин-макс значения) 

## Зачем
1. Поработать с ООП.
2. Попрактиковать паттерны.
3. Познакомиться с "низким" уровнем.
